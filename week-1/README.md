# Nädal 1: SQL Basics

## Projekti eesmärk

Selle nädala eesmärk oli tutvuda SQL-i põhikäskudega ning õppida uurima UrbanStyle ettevõtte andmeid. Analüüsi eesmärk oli hinnata andmete kvaliteeti ja leida võimalikke probleeme enne põhjalikuma ärianalüüsi tegemist.

## Mida ma tegin

Minu roll meeskonnas oli **Roll A – Sales**.

Tegin järgmised tegevused:
- uurisin `sales` tabelit SQL päringutega;
- kontrollisin müügikirjete koguarvu;
- leidsin suurimad ja väikseimad müügitehingud;
- kontrollisin puuduvaid `customer_id` väärtusi;
- analüüsisin müügiandmete kvaliteeti;
- osalesin meeskonna ühise Data Landscape kokkuvõtte koostamisel.

## Tulemused

Analüüsi käigus selgus, et:
- müügitabelis oli **15 234** kirjet;
- suurim müügitehing oli **2170,40 €**;
- väikseim müügitehing oli **-1405,32 €**;
- **1487** müügikirjel puudus kliendi ID.

## Äriline mõju

Andmete kvaliteedi kontroll aitab tagada usaldusväärsemad analüüsid ja paremad ärilised otsused. Enne põhjalikuma analüüsi tegemist tuleks uurida negatiivseid müügisummasid ning puuduvaid kliendiandmeid.

## Mida õppisin

Õppisin kasutama SQL-i põhikäskusid (`SELECT`, `WHERE`, `ORDER BY`, `COUNT`, `MIN`, `MAX`) ning sain parema arusaama sellest, kuidas hinnata andmete kvaliteeti enne nende analüüsimist.
